Свойства атрибута animation

Здравствуйте! Поскольку на официальном сайте и других ресурсах нет качественной документации, я решил создать её самостоятельно.

В этой статье я подробно описываю все свойства атрибута animation. Здесь вы найдёте всё необходимое.

  • autoplay определяет, должна ли анимация начинаться автоматически при загрузке сцены. Значение по умолчанию - true.
  • delay задает задержку перед началом анимации, указанную в миллисекундах.
  • direction задаёт направление анимации, например, "normal", "reverse", "alternate".
  • dur определяет продолжительность анимации в миллисекундах.
  • easing устанавливает функцию плавности для анимации (например, "linear", "ease-in", "ease-out").
  • easing определяет, как будет изменяться скорость анимации в течение времени. Это позволяет создавать более естественные и плавные переходы, а не линейные изменения. Существует несколько типов easing-функций, которые можно использовать:
    • Linear - анимация происходит с постоянной скоростью.
    • Ease In - анимация начинается медленно и ускоряется.
    • Ease Out - анимация начинается быстро и замедляется к концу.
    • Ease In Out - анимация медленно начинается, ускоряется в середине и снова замедляется к концу.
  • easing - тип функции сглаживания, которая определяет, как анимация будет ускоряться и замедляться (например, linear, ease-in, ease-out и т.д.).
  • fade позволяет задать параметры для эффектов плавного перехода, например, для изменения прозрачности объекта.
  • fill определяет, как анимация будет вести себя после завершения (например, none, forwards, backwards, both).
  • from задаёт начальное состояние анимации.
  • loop позволяет задать, должна ли анимация повторяться. Если установить на true, анимация будет повторяться после окончания.
  • mixin позволяет указать набор анимаций, которые могут быть применены к объекту.
  • on указывает, на каком событии анимация должна запускаться (например, mouseenter, mouseleave).
  • once позволяет установить, должна ли анимация срабатывать только один раз или снова, если она завершится.
  • onEnd задаёт обработчик события, который будет вызван по завершении анимации. Например, для выполнения какого-то действия после окончания анимации.
  • onPause определяет обработчик события, который вызывается при приостановке анимации.
  • onResume определяет обработчик события, который вызывается после возобновления анимации.
  • onStart задаёт обработчик события, который будет вызван в начале анимации. Это может быть полезно для выполнения дополнительных действий при запуске анимации.
  • pause позволяет приостановить анимацию в определённый момент. Это может быть полезно, если нужно остановить анимацию на каком-либо кадре.
  • pause указывает, должна ли анимация быть приостановлена при загрузке (по умолчанию false).
  • pauseEvents - события, которые приостанавливают анимацию.
  • position позволяет задать изменения позиции 3D-объекта во время анимации.
  • property указывает, какое свойство объекта будет анимироваться (например, position, rotation, scale и т.д.).
  • repeat устанавливает количество раз, которые анимация должна повториться. По умолчанию 1, можно установить значение infinite для бесконечного воспроизведения.
  • repeat указывает, сколько раз анимация будет повторяться. Можно использовать true для бесконечного повторения.
  • resumeEvents - события, которые возобновляют анимацию.
  • rotation задаёт параметры для вращения объекта.
  • scale управляет масштабированием объекта во время анимации.
  • start задаёт время, когда анимация должна начинаться, специфично для определенного свойства.
  • startEvents - события, которые запускают анимацию (например, click, mouseenter и т.д.).
  • stop позволяет указать время, когда анимация должна быть остановлена.
  • to определяет конечное состояние анимации, указывая значения для таких свойств, как позиция, масштаб, вращение и т.д.
  • to можно использовать для указания конечного значения анимации в виде строки, например, to: '0 1 0' для изменения позиции.
